Questions and Answers

Q: Difference between map, filter, forEach, find?

map: map creates a new array with the results of calling a provided function on every element in the array. It returns a new array with the same length as the original array.

filter: filter creates a new array with all elements that pass the test implemented by the provided function. It returns a new array with only the elements that pass the test.

forEach: forEach executes a provided function once for each array element. It does not return anything and only executes the callback function for each element.

find: find returns the value of the first element that satisfies the provided function, otherwise, it returns undefined.

Q: Difference between localStorage, SessionStorage, Cookie?

localStorage: Data stored in localStorage is persistent and will remain even after the browser is closed or the computer is turned off.

sessionStorage: sessionStorage is similar to localStorage, but the data stored in sessionStorage is only available for the duration of the current browser session.

Cookie: Cookies are small text files that are stored in a user's browser by a website. Cookies can be used to store user preferences, login information, and other data. Cookies can have an expiration date, after which they will be automatically deleted by the browser.

Q: How the event loop Work?

The event loop is a mechanism that allows the JavaScript engine to handle asynchronous code by continuously checking for and executing tasks that are waiting in a queue. When an asynchronous task is started, it is added to the queue, and the JavaScript engine continues to execute the rest of the code. When the task is completed, a callback function is added to the queue to handle the result.
